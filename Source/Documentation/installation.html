Installing

The OS9exec emulator is called os9App for MacOS9, os9App_univ for MacOSX, os9App.exe for Windows or os9linux for Linux.

Installation for Linux

  1. Put the OS9exec program somewhere where it will be in the current path.
  2. Create a base directory.
  3. The base directory must be either:
    • a directory called dd which contains a directory, dd/CMDS, that must contain the original OS-9 binaries.
    • a 1:1 RBF raw image from an original OS-9 system as a file called dd.
    The performance of the RBF emulation is usually better and sometimes more compliant for tricky things. Nevertheless you can always create an empty RBF Image file and copy files into it. Or use the RamDisk /r0, which is also a normal RBF RamDisk.
  4. If a file called startup is in dd, it will be called at the beginning. Try the existing /dd/startup. It might work as it stands, so try it. There is a built-in debugger system which you can call with os9linux -d 3 shell to get more info about what's going on.
  5. The host's Mac/Windows/Linux directory paths can be accessed directly. Relative paths work as expected. Files and directories below the dd directory can be referred to as /dd/..., but any host's files or directories outside the dd directory structure must be referred to by absolute name until mount is extended to allow them to be mounted as pseudo devices.

    NOTE (for Linux/MacOSX): In the mean time, if you have root privileges you can put a symbolic link in the root (/) directory that points to the directory you want to reference:

    ln -s /home/osk/fd0 fdx

    allows you the refer to the directory as either /home/osk/fd0 or as /fdx from inside OS9exec. However this is not recommended because of possible security breaches.
